# CountStud

Вынести хэширование паролей в отдельную директорию. 
Сделать создание структуру пользваотелей админ\учитель
добавить новое взаимодействие с БД
Сделать новую таблицу
Связать админ и табл со студентами по ID


1) Спроектировать структуру Users. Она будет включать в себя users(преподователи)/admin(администратор системы)
2) Написать новое взаимодействие с БД, а то есть, получить студента, удалить(из группы), получить всех студентов своей группы
3) Добавить хэширование паролей для БД с препод\админ.
4) Добавить JWT токен и работу с midllware
5) Реализовать аутентификацию (логин пользователя и выдача токена)
6) Реализовать авторизацию на основе ролей (admin / teacher)
7) Добавить проверку принадлежности студента преподавателю (access control)
8) Реализовать CRUD-операции для пользователей (создание преподавателей администратором)
9) Добавить валидацию входных данных (login, password, id и т.д.)
10) Реализовать единый формат обработки и возврата ошибок
11) Добавить конфигурацию приложения (.env: БД, JWT_SECRET и др.)
12) Организовать структуру проекта (handler / service / repository / middleware)
13) Добавить логирование основных действий и ошибок
14) Ограничить доступ к методам через middleware (защищённые маршруты)
